不允许删除系统文件。

## 全局构建约束

- 项目自身的构建产物、打包中间产物、发布输出、测试报告等，默认收口到项目根目录的 `build/` 或项目既有构建输出目录下。
- 开发语言、包管理器、编译器自身的全局缓存，允许使用其标准缓存目录和安装目录，不需要为每个项目单独重定向到 `build/`。例如：
  - Go: 默认 `GOMODCACHE`、`GOCACHE`
  - Node/npm/pnpm/yarn: 默认 cache/store
  - Composer、Gradle、Flutter/Pub、Cargo 等工具的默认缓存
- 不要为了缓存隔离而手动把语言级依赖缓存复制、下载或重定向到每个项目内，除非用户明确要求，或项目本身已有这类配置。
- 临时文件优先使用工具默认行为；只有明确生成项目产物或中间产物时，才收口到项目 `build/`。
- 清理时只清理当前项目明确生成的构建产物；不要删除系统目录、用户主目录缓存目录或语言工具全局缓存，除非用户明确要求。

## pte-live-im 项目边界

- 本项目是独立 IM 项目集合，只包含 IM API、WebSocket、gRPC、聊天业务、IM 管理后台与 MQ 相关服务。
- Docker Compose 默认只启动 `api-im` 和 `mq-pulsar`。
- 本项目定义统一 Docker 网络 `pte_live_net`，子网 `172.30.0.0/24`，网关 `172.30.0.1`。
- 本仓库不包含外部商城 API 或外部业务项目代码。
- 不要让 IM 代码硬编码外部业务系统数据库、发布目录或内部实现；外部系统只能通过 HTTP / gRPC / WebSocket / MQ 协议接入 IM。
- 项目相关域名统一使用 `ptelive.com`，IM 入口默认 `im.ptelive.com`。
